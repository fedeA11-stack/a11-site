## Executive Summary

This audit supersedes the [2026-06-19 plan](a11-seo-agent-plan-2026-06-19.md), which scored the site **32/100**. Nearly that entire plan has since shipped: `metadataBase`, `robots.ts`, `sitemap.ts`, per-page metadata on every route, 10 per-case OpenGraph images, `manifest.ts`, Organization + BreadcrumbList + CreativeWork JSON-LD, a 308 redirect for the renamed route, immutable asset caching, and `next/font` + `next/image` hygiene. The build is clean and all 33 routes prerender as static HTML — crawlers and AI agents get full content with no client-fetch dependency.

The remaining issues are **not** missing infrastructure — they are **data drift** in the one hand-maintained surface (`public/llms.txt`) and a couple of small completeness gaps. Three of the findings are outright bugs: a wrong contact email, a stale rebranded entry pointing at a redirected URL, and a missing case study. None are hard to fix; all are worth fixing because `llms.txt` is the file AI agents read first.

Because this is a studio portfolio, the e-commerce dimensions of a standard audit (keyword research, competitor gap analysis, buyer-intent content calendars) **do not apply** and are intentionally omitted. The honest growth lever here is off-codebase: Search Console verification + a small amount of authority building.

## Top 5 Critical Actions

1. **Fix the wrong contact email in `public/llms.txt`.** It reads `hello@a11studio.com`; the source of truth (`app/seo.ts`) is `hello@a11.studio`. An AI agent quoting this would hand out a non-existent address. *(Bug — 1 line)*
2. **Fix the stale "TokenStudio" entry in `llms.txt`.** It still calls the project "TokenStudio" and links `https://a11.studio/tokenstudio`, which now 308-redirects to `/freehold-invest`. Rename to "Freehold Invest" and point at the live URL. *(Bug — 2 fields)*
3. **Add the missing "Nous" case study to `llms.txt`.** `ALL_PROJECTS` has 10 projects; `llms.txt` lists 9. Nous is in the sitemap and has a live page but is invisible to agents reading the index. *(Bug — 1 entry)*
4. **Add `sameAs` to the Organization JSON-LD** in `app/layout.tsx`, sourced from `app/socialLinks.ts` (real X + LinkedIn URLs already exist). This is the standard signal that ties the site to its social profiles for entity/knowledge-graph resolution. *(Gap — low effort)*
5. **Stop `llms.txt` from drifting again.** All three bugs above exist because `llms.txt` is hand-written while every other surface derives from `caseProjects.ts` + `seo.ts`. Either generate it from those sources (a route handler at `app/llms.txt/route.ts`, mirroring how `sitemap.ts` works) or add a one-line test asserting it covers every `ALL_PROJECTS` href. *(Root cause — prevents recurrence)*

## Detailed Findings

### Technical SEO — 92/100 (strong)
- `metadataBase: new URL("https://a11.studio")` set in `app/layout.tsx`; all OG/canonical URLs resolve absolutely.
- `app/sitemap.ts` covers all 5 top-level routes + 10 case studies derived from `ALL_PROJECTS` (no drift).
- `app/robots.ts` allows all and references the sitemap. **Minor:** includes a non-standard `host` directive (only Yandex honors it) — harmless noise.
- `next.config.ts`: 308 redirect `/tokenstudio → /freehold-invest`, `X-Powered-By` disabled, AVIF/WebP, 1-year immutable caching on `/assets`.
- Clean `next build`: 33 routes, all `○ (Static)` prerendered.

### Metadata & Social — 88/100 (strong)
- Root template `%s · A11 Product Studio`; per-page `metadata` on every route except home.
- `buildCaseMetadata()` derives title/description/canonical/OG/Twitter from each case's typed `data` object — drift-free by construction.
- 10 per-case `opengraph-image.tsx` segments generate share images at build time.
- **Gaps:** `/world`, `/studio`, `/manifesto`, `/contact` fall back to the root OG image. Home (`app/page.tsx`) is still a client component and inherits the layout default metadata (acceptable, since the default *is* the home/Work copy).

### Structured Data / Agent Legibility — 74/100 (the weak spot)
- Organization JSON-LD sitewide (`app/layout.tsx`); BreadcrumbList + CreativeWork `@graph` per case study (`app/CaseStudy.tsx` via `caseStudyJsonLd`). Well-structured.
- **Missing `sameAs`** on the Organization node despite real X + LinkedIn URLs in `socialLinks.ts`.
- **`public/llms.txt` is stale and buggy** — the only hand-maintained surface, and it has drifted:
  - Contact email `hello@a11studio.com` ≠ source-of-truth `hello@a11.studio`.
  - "TokenStudio" entry uses the pre-rebrand name and the redirected `/tokenstudio` URL (should be "Freehold Invest" / `/freehold-invest`).
  - Missing the Nous case study (9 of 10 projects listed).
  - Missing `/contact`.

### Content & Semantics — 85/100 (good)
- One `<h1>` per page (case title / page hero); section titles are real `<h2>`s in `CaseStudy.tsx`; mobile cards use `sr-only` h2s where the visible mark is a decorative wordmark.
- Alt text is descriptive on meaningful images and correctly empty (`alt="" aria-hidden`) on decorative wordmarks/SVGs.
- Each case study has a unique, human-written `description` reused consistently across metadata, OG, and JSON-LD.

## What the Site Does Well
- **Fully static (SSG):** every route ships complete HTML — ideal for crawlers and AI agents.
- **Single sources of truth:** `seo.ts`, `caseProjects.ts`, `socialLinks.ts` keep sitemap/metadata/JSON-LD from drifting. (The one file *outside* this discipline — `llms.txt` — is exactly where the bugs are.)
- **Performance won't fight SEO:** `next/font/local` with `display:swap` (no CLS), `next/image` with static imports + blur placeholders, AVIF-first, immutable asset caching.
- **Migration safety:** the rename was handled correctly with a permanent redirect.
- **Accessibility-aware:** `prefers-reduced-motion` respected; semantic headings; sensible alt discipline.

Standard e-commerce SEO dimensions — keyword research, competitor gap analysis, product/category-page optimization, buyer-intent content calendars — were intentionally **not** run: a11.studio is a studio portfolio, not a store, so those frameworks would produce noise. The real growth lever beyond this codebase is named-entity authority (Search Console, complete social/directory profiles), reflected in the 31–90 day section.
